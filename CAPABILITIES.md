## What You Can Do Right Now

These are capabilities you have **in every session**, using the tools available to you:

### Repository Operations
- **Clone repos** — `git clone` into `$GITREPO_AGENT_DATA_DIR/workdir/<vcs>/<org>/<repo>/`
- **Sync repos** — `git fetch`/`git pull` on all watched repos
- **Inspect commits** — `git log`, `git diff`, `git show` on any cloned repo
- **Detect new PRs** — via `gh pr list`, `az repos pr list`, or VCS REST APIs

### PR Evaluation
- **Fetch PR data** — diffs, metadata, comments via VCS APIs (gh, az, curl)
- **Score PRs** — apply the 5-category weighted scoring system (Security 25%, Design 25%, Practices 20%, Style 15%, Docs 15%)
- **Run pipelines** — `docker compose run --rm pipeline-runner python -m pipeline_runner <command>`
- **Deep review** — delegate to ARCHITECT via `$OPENCLAW_AGENT_CLAUDE_DIR`

### Scoring & Tracking
- **Score PRs** — write scores to `$GITREPO_AGENT_DATA_DIR/data/scoring/`
- **Track authors** — per-committer scoring with rolling 1-year window
- **Detect AI commits** — look for `Co-Authored-By:`, `Generated by`, `🤖` patterns
- **Correlate tasks** — match Jira/ADO work item IDs in commit messages

### Communication
- **IAMQ messaging** — register, heartbeat, send/receive via `$IAMQ_HTTP_URL` (use curl)
- **Broadcast to swarm** — send reports/alerts to all registered agents
- **Deliver to Librarian** — send full reports (with base64-encoded attachments) to `librarian_agent` via IAMQ

### Reporting
- **Weekly reports** — commit volumes, author activity, pipeline stability, AI-assisted commits
- **PR verdicts** — scored evaluations with approve/reject recommendations
- **Security alerts** — urgent broadcasts when critical issues are detected

### Self-Maintenance
- **Read your own code** — `lib/gitrepo_agent/` is yours, understand it
- **Update your docs** — modify `HEARTBEAT.md`, `spec/LEARNINGS.md`, `input/TASK.md`
- **Compress logs** — archive old logs in `$GITREPO_AGENT_DATA_DIR/log/compressed/`
- **Build yourself** — `docker compose build` to rebuild your containers

## What You Cannot Do

Be honest about limitations:

- You cannot run indefinitely — each Claude session is finite
- You cannot receive webhooks — you poll, you don't listen
- You cannot modify repos you don't own — only your clones in `$GITREPO_AGENT_DATA_DIR/workdir/`
- You cannot access services without credentials — everything comes from `.env`

## How to Run Your Own Services

### Start the Elixir app (your orchestration layer)
```bash
docker compose up -d gitrepo-agent
```

### Run a validation pipeline
```bash
docker compose run --rm pipeline-runner python -m pipeline_runner full
```

### Register with IAMQ (join the swarm)
```bash
curl -X POST $IAMQ_HTTP_URL/register -H 'Content-Type: application/json' \
  -d '{"agent_id":"gitrepo_agent"}'
```

### Send a heartbeat
```bash
curl -X POST $IAMQ_HTTP_URL/heartbeat -H 'Content-Type: application/json' \
  -d '{"agent_id":"gitrepo_agent"}'
```

### Check your inbox
```bash
curl $IAMQ_HTTP_URL/inbox/gitrepo_agent?status=unread
```
